## Детейлинг TMA — MVP

Это минимальный рабочий прототип сервиса бронирования в детейлинг-центр:
- backend на Node.js + Express
- frontend на React + Vite
- тёмный, строгий интерфейс с акцентами и мягким «сиянием»

### Что тебе понадобится

- Node.js (LTS, примерно 18+)
- npm (ставится вместе с Node.js)

### Структура проекта

- `backend` — сервер с API (услуги, создание брони, список броней)
- `frontend` — веб-приложение (интерфейс Mini App, форма записи)

### Как запустить

1. **Backend**
   ```bash
   cd backend
   npm install
   npm run dev
   ```
   Сервер поднимется на `http://localhost:4000`.

2. **Frontend**
   Открой новое окно/вкладку терминала:
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
   Vite поднимет фронт на `http://localhost:5173` (точный адрес он напишет в консоль).

3. **Проверка работы**
   - Открой в браузере адрес, который покажет Vite (обычно `http://localhost:5173`).
   - Должен открыться тёмный интерфейс:
     - выбор услуги (оклейка, мойка, шины)
     - выбор даты и времени
     - ввод авто и телефона
     - кнопка «Записаться»
   - После отправки формы должна прийти успешная отметка, а на backend появится новая бронь (пока хранится в памяти, пока сервер запущен).

4. **(Опционально) Запуск бота локально**

   Для бота нужны переменные окружения:

   - `BOT_TOKEN` — токен бота из `@BotFather`
   - `WEBAPP_URL` — публичный URL фронтенда (когда задеплоишь, например `https://tma.your-domain.com`)

   Пример для PowerShell (Windows), перед запуском бота:

   ```powershell
   $env:BOT_TOKEN="ТВОЙ_ТОКЕН_ИЗ_BOTFATHER"
   $env:WEBAPP_URL="https://tma.your-domain.com"
   cd backend
   npm run bot
   ```

   Локально бот будет общаться с Telegram через long polling (без webhook).

### Telegram Mini App (бот)

Чтобы связать это с Telegram:

1. Создай бота через `@BotFather` и получи токен.
2. В backend можно добавить отдельный скрипт бота (я приведу пример кода ниже), который:
   - показывает кнопку «Открыть приложение»;
   - открывает твою веб-страницу как Mini App (когда ты задеплоишь фронт на публичный URL).

Для локальной проверки достаточно, чтобы у тебя работали `http://localhost:4000` и `http://localhost:5173`. Telegram-часть требует публичного адреса, поэтому её ты проверишь уже после выкладки.

### Telegram Mini App (общая идея)

- Интерфейс, который ты видишь в браузере, можно встроить в Telegram как Mini App (WebApp).
- Для этого нужно будет:
  1. Задеплоить frontend + backend на внешний сервер (чтобы был публичный URL).
  2. В `@BotFather` указать URL мини-аппы.
  3. В коде бота отправлять кнопку, которая открывает этот URL как WebApp.

В этом MVP я настрою:
- API, которое уже готово для использования из Mini App.
- Фронт, который умеет работать как обычный сайт и как TMA (через Telegram WebApp SDK, если он есть в окружении).

### Деплой как TMA (краткий план)

Ниже — общий сценарий, более детально я распишу тебе шаги:

1. Создать аккаунты на GitHub, Vercel (frontend) и Render (backend).
2. Залить этот проект в репозиторий GitHub.
3. Подключить репозиторий к Render и задеплоить `backend`:
   - build: `npm run build`
   - start: `npm run start`
   - переменные окружения: `ALLOWED_ORIGIN` (URL фронта).
4. Подключить репозиторий к Vercel и задеплоить `frontend`:
   - root: `frontend`
   - build: `npm run build`
   - output: `dist`
   - переменная окружения: `VITE_API_BASE` (URL backend’а с Render).
5. В `@BotFather` указать `WEBAPP_URL` (URL фронтенда) и включить WebApp-кнопку.
6. Запустить бота локально с `BOT_TOKEN` и `WEBAPP_URL`, чтобы по `/start` приходила кнопка «Открыть запись».
